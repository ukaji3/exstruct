# v0.2.61 Release Notes

This release standardizes error handling across extraction, serialization, and
rendering, and documents coverage expectations for tests.

## Highlights

- Exception hierarchy: added `ExstructError` plus domain-specific errors
  (`ConfigError`, `ExtractionError`, `SerializationError`,
  `MissingDependencyError`, `RenderError`, `OutputError`, `PrintAreaError`) for
  consistent error handling. See `src/exstruct/errors.py`.
- Clear dependency guidance: unsupported formats raise `SerializationError`, and
  missing optional dependencies raise `MissingDependencyError` with installation
  guidance. See `src/exstruct/io/__init__.py` and `src/exstruct/render/__init__.py`.
- Auto page-break behavior: `export_auto_page_breaks` now raises
  `PrintAreaError` when no auto page-break areas are present. See
  `src/exstruct/__init__.py`.
- Test requirements spec: published v0.2 coverage expectations for extraction,
  output, CLI, and error handling. See `docs/agents/TEST_REQUIREMENTS.md`.

## Compatibility Notes

- `PrintAreaError` inherits `ValueError`, so existing handlers still work, but
  using the new type makes intent explicit.
- If your code calls `export_auto_page_breaks`, handle `PrintAreaError` or verify
  that `auto_print_areas` is populated before exporting.
